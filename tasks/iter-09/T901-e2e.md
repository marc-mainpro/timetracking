# T901 — Test end-to-end del flujo completo

- Iteración: 9 · Depende de: T704, T801 · Contexto: CONTEXT-GLOBAL, SDD §19 (E2E)

## Objetivo
Test E2E automatizado del flujo mínimo de la sección 19 del SDD, contra el backend real.

## Detalle
1. Implementar como test de integración de API de alto nivel (`EndToEndFlowIT`, Testcontainers + RestAssured/WebTestClient) — no requiere navegador; el E2E de UI queda fuera del MVP (documentar).
2. Flujo en orden, con aserciones en cada paso: (1) registrar tenant → (2) el admin queda creado (login admin) → (3) admin crea empleado → (4) login empleado → (5) inicia jornada → (6) inicia pausa → (7) finaliza pausa → (8) finaliza jornada → (9) consulta historial y ve la jornada → (10) solicita corrección → (11) admin la aprueba → (12) existe registro de auditoría de la aprobación → (13) existen mensajes outbox de los eventos del flujo y el publicador los publica.
3. Segundo escenario: repetir con un segundo tenant en paralelo y verificar aislamiento total de datos (criterio global "dos tenants operan sin compartir datos").

## Criterios de aceptación
- `mvn verify` verde con el E2E incluido en CI (si es lento, perfil/etiqueta `e2e` ejecutada en CI igualmente).
